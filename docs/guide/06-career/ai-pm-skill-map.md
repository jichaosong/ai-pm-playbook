## 硬技能

### 1. LLM 原理

| 层级 | 技能点 | 掌握标准 |
|------|--------|---------|
| L1 了解 | Transformer 基本架构 | 能解释 Attention 机制和自回归生成的基本原理 |
| L1 了解 | Token 与 Tokenizer | 理解 Token 化过程，知道不同模型 Token 计费方式 |
| L1 了解 | 训练三阶段 | 能区分预训练、SFT、RLHF 的目的和区别 |
| L2 掌握 | RAG 原理 | 理解检索增强生成的全流程：Embedding → 检索 → 生成 |
| L2 掌握 | Fine-tuning 场景 | 知道什么时候该用 Fine-tuning 而非 Prompt Engineering |
| L2 掌握 | Agent 架构 | 理解 ReAct、Function Calling、Tool Use 等 Agent 模式 |
| L3 精通 | 模型推理优化 | 理解 KV Cache、量化（INT8/INT4）、Speculative Decoding 等 |
| L3 精通 | 成本模型 | 能估算模型训练/推理成本，做技术选型的 ROI 分析 |
| L3 精通 | 多模态理解 | 了解多模态模型（视觉、语音、视频）的工作原理和产品整合方式 |

### 2. 提示词工程

| 层级 | 技能点 | 掌握标准 |
|------|--------|---------|
| L1 了解 | 基础 Prompt 编写 | 能写出清晰的指令，获得期望的输出格式 |
| L1 了解 | Role Prompting | 能通过角色设定提升输出质量 |
| L2 掌握 | Few-shot Prompting | 能在 Prompt 中加入示例来引导模型输出 |
| L2 掌握 | Chain-of-Thought | 能设计思维链 Prompt，提升复杂推理任务的表现 |
| L2 掌握 | System Prompt 设计 | 能为产品设计完整的 System Prompt，包含角色、规则、约束 |
| L2 掌握 | Prompt 版本管理 | 使用工具管理 Prompt 版本，记录每次变更的效果 |
| L3 精通 | 复杂 Prompt Pipeline | 能设计多步骤、多分支的 Prompt Chain |
| L3 精通 | Prompt 自动化评估 | 搭建自动化的 Prompt 评测流程，用指标驱动优化 |
| L3 精通 | 对抗性 Prompt 防御 | 了解 Prompt Injection 攻击方式，设计防御策略 |

### 3. 模型评估

| 层级 | 技能点 | 掌握标准 |
|------|--------|---------|
| L1 了解 | 基础评估指标 | 理解准确率、精确率、召回率、F1 Score 的含义 |
| L1 了解 | 人工评估 | 能设计人工评估标注规范和评分标准 |
| L2 掌握 | 自动化评估 | 掌握 BLEU/ROUGE/METEOR 等自动化评估方法及其局限性 |
| L2 掌握 | LLM-as-Judge | 能用 GPT-4/Claude 等模型做自动评估，设计评估 Prompt |
| L2 掌握 | 评估数据集 | 能构建高质量评估数据集（Golden Dataset），包含边界情况 |
| L3 精通 | 全面评估框架 | 搭建包含准确性、安全性、公平性、鲁棒性的多维评估体系 |
| L3 精通 | 评估 Pipeline | 构建 CI/CD 中的自动评估 Pipeline，与开发流程集成 |
| L3 精通 | 红队测试 | 组织并分析红队测试结果，识别模型的安全和伦理风险 |

### 4. 数据分析

| 层级 | 技能点 | 掌握标准 |
|------|--------|---------|
| L1 了解 | SQL 基础 | 能写 SELECT、JOIN、GROUP BY 做数据查询和统计 |
| L1 了解 | 指标定义 | 能定义产品核心指标（DAU、留存率、转化率等） |
| L2 掌握 | A/B 测试 | 能设计 A/B 测试方案，理解统计显著性、样本量计算 |
| L2 掌握 | 数据分析工具 | 能使用 Python/Pandas 或 BI 工具（Metabase/Tableau）做分析 |
| L2 掌握 | 用户行为分析 | 能通过用户行为数据发现产品问题和优化机会 |
| L3 精通 | 因果推断 | 理解 DID、PSM 等因果推断方法在 AI 产品评估中的应用 |
| L3 精通 | 指标体系建设 | 建立 AI 产品 OKR 到关键指标的完整拆解体系 |
| L3 精通 | AI-specific 指标 | 能定义和追踪幻觉率、上下文利用率、调用成本等 AI 特有指标 |

## 软技能

### 1. 技术沟通

| 技能点 | 说明 |
|--------|------|
| **与算法工程师沟通** | 能用准确的技术语言表达产品需求；理解算法工程师的工作方式和约束 |
| **与非技术团队沟通** | 能用业务语言解释 AI 能力边界，管理非技术干系人的期望 |
| **技术文档** | 能写出算法团队可执行的 PRD，包含明确的输入输出规范、评估标准 |
| **需求拆解** | 能将模糊的产品需求拆解为清晰的技术任务和验收条件 |
| **技术方案理解** | 能阅读技术设计文档并提出有建设性的产品视角意见 |

### 2. 跨团队协作

| 技能点 | 说明 |
|--------|------|
| **敏捷管理** | 能组织 AI 产品的 Sprint 规划、站会、复盘等敏捷活动 |
| **冲突解决** | 能在资源有限的情况下协调多个团队的优先级冲突 |
| **标注团队管理** | 能与数据标注团队协作制定标注规范，评估标注质量 |
| **设计协作** | 能配合 UX 设计师设计 AI 产品的交互形态，特别是不确定性展示 |
| **项目管理** | 能推动跨团队项目按时交付，识别和化解项目风险 |

### 3. 快速学习

| 技能点 | 说明 |
|--------|------|
| **论文阅读** | 能快速阅读 AI 论文，提取关键的产品启示 |
| **技术实验** | 能动手体验新模型/新工具，快速验证产品想法 |
| **持续跟踪** | 建立信息获取渠道（Newsletter、Twitter/X、Papers_with_Code）|
| **从数据中学习** | 能从用户数据和模型评估数据中发现学习机会 |
| **知识体系搭建** | 能建立自己的 AI 知识库（第二大脑/笔记系统） |

## 工具技能

### 1. AI 工具链

| 工具类别 | 代表工具 | 掌握程度 |
|---------|---------|---------|
| **LLM API** | OpenAI API, Claude API, Gemini API | 能用 API 完成原型验证和测试 |
| **开发框架** | LangChain, LlamaIndex, Haystack | 理解基本概念，能搭建简单的 RAG Demo |
| **向量数据库** | Pinecone, Milvus, Weaviate | 了解向量检索原理和不同方案优劣 |
| **模型平台** | Hugging Face, Replicate | 能搜索、比较和测试开源模型 |
| **AI 应用平台** | Dify, Coze, Gradio | 能用低代码平台快速搭建原型 |
| **AI 编码** | Cursor, Claude Code, Copilot | 使用 AI 辅助提升工作效率 |

### 2. Prompt 工具

| 工具 | 用途 | 推荐场景 |
|------|------|---------|
| **Anthropic Console** | Prompt 编写和测试 | 设计 Claude 的 System Prompt，内置评估功能 |
| **OpenAI Playground** | GPT 模型 Prompt 测试 | 快速迭代和对比不同 Prompt 的效果 |
| **LangSmith** | Prompt 版本管理与评估 | 团队协作的 Prompt 全生命周期管理 |
| **PromptPerfect** | Prompt 优化 | 自动优化和调试复杂 Prompt |
| **Portkey** | Prompt 网关和监控 | 生产环境 Prompt 的监控和成本管理 |
| **Agenta** | 开源 Prompt 管理平台 | 团队内部的 Prompt 协作和评估 |

### 3. 评估平台

| 平台 | 用途 | 推荐场景 |
|------|------|---------|
| **LangSmith** | LLM 应用评估和调试 | 端到端的 Trace 和评估 |
| **Weights & Biases** | 实验追踪和可视化 | 模型 Fine-tuning 和 Prompt 实验跟踪 |
| **MLflow** | 机器学习生命周期管理 | 模型版本管理和评估指标记录 |
| **G-Eval / RAGAS** | RAG 系统评估 | 专门评估 RAG 系统的检索和生成质量 |
| **HumanLoop / Scale AI** | 人工标注和评估服务 | 大规模人工评估的标签订阅服务 |
| **自定义评估 Dashboard** | 内部搭建评估看板 | 针对特定产品的定制化评估指标监控 |

## 技能成长路线

### 第一阶段（L1 入门）关注
```
硬技能 ⊛ LLM 基础概念 + 基础 Prompt + 基础数据分析
软技能 ⊛ 技术文档写作 + 任务跟进
工具   ⊛ 会使用 ChatGPT/Claude + 基础 SQL
```

### 第二阶段（L2 独立）关注
```
硬技能 ⊛ RAG/Fine-tuning + 进阶 Prompt + 自动化评估
软技能 ⊛ 技术沟通 + 跨团队协调 + 主动学习
工具   ⊛ LangChain + Prompt 工具 + 评估工具
```

### 第三阶段（L3 资深）关注
```
硬技能 ⊛ 推理优化 + 成本模型 + 全面评估体系
软技能 ⊛ 跨团队管理 + 技术判断 + 战略思维
工具   ⊛ 工具链选型 + 评估 Pipeline + 指标看板
```

### 第四阶段（L4 专家）关注
```
硬技能 ⊛ 行业洞察 + 技术趋势判断
软技能 ⊛ 组织建设 + 影响力 + 商业思维
工具   ⊛ 标准制定 + 方法论输出 + 生态建设
```
