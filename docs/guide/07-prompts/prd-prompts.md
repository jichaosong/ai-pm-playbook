## 模板 1：通用 AI 功能 PRD

**使用场景**：为一个现有产品增加 AI 功能（如智能搜索、智能推荐、智能分类）。需要和算法团队、工程团队对齐需求。

**提示词**：

```markdown
你是一位资深 AI 产品经理。请帮我写一份 PRD（产品需求文档）。

## 产品背景
[在这里描述你的产品和业务背景]

## 要做的 AI 功能
[在这里描述你想加的 AI 功能]

## 目标用户
[在这里描述目标用户]

## 请包含以下内容：

### 1. 概述
- 产品背景和为什么需要这个 AI 功能
- 成功标准（OKR / KPI — 至少要包含 2 个量化目标）
- 约束条件（预算、时间、资源）

### 2. 用户需求
- 写 3-5 个用户故事（User Story），格式：As a [角色], I want to [目标], so that [价值]
- 优先级标注（P0/P1/P2）

### 3. 功能需求
- 用户使用流程（用文字描述，包含主要路径和异常路径）
- 输入规范：用户输入的数据格式和范围
- 输出规范：AI 返回的数据格式和展示方式
- 边界条件：至少列出 3 个边界情况和处理方式
- 错误处理：模型不可用、超时、幻觉输出的处理方案

### 4. AI 技术需求
- 模型选型建议（GPT-4 / Claude / 开源模型 / 自研模型）
- 数据需求（需要哪些数据来支撑这个功能）
- Prompt 方向建议（如果需要用 LLM）
- 性能要求（响应时间 < X 秒，准确率 > X%）

### 5. 评估标准
- 核心评估指标（至少 3 个）
- 评估数据集要求
- 验收条件：明确的 Pass/Fail 标准

### 6. 灰度上线计划
- 灰度策略（按用户比例 / 按内测用户）
- 监控指标
- 回滚方案

### 7. 风险和缓解措施
- 列出 3 个以上潜在风险
- 每个风险的缓解措施

请用专业的 PRD 格式输出，语言简洁清晰，可以直接用于与算法和工程团队沟通。
```

## 模板 2：RAG 问答系统 PRD

**使用场景**：基于企业知识库或私有数据做问答系统（如智能客服、内部知识问答、文档问答）。

**提示词**：

```markdown
你是一位专注于 RAG（检索增强生成）系统的 AI 产品经理。请帮我写一份 RAG 问答系统的 PRD。

## 背景信息
- 产品名称：[填写产品名]
- 知识库内容范围：[如：公司内部 SOP、产品文档、FAQ]
- 知识库规模：[如：约 500 篇文章、总计约 200 万字]
- 目标用户：[如：客服人员、内部员工、外部客户]
- 核心场景：[如：7×24 小时回答常见问题]

## 请输出包含以下内容的 PRD：

### 1. 产品目标和成功指标
- 核心目标（如：首次响应时间 < 5 秒、回答准确率 > 85%）
- 次要指标（如：用户满意度评分、人工转接率、覆盖率）

### 2. RAG 架构需求
- 文档处理流程（上传 → 解析 → 分块 → Embedding → 存储）
- 分块策略建议（chunk size、overlap）
- 检索策略建议（Hybrid Search / 多路召回 / 重排序）
- 生成策略（直接回答 / 引用回答 / 不确定时转人工）
- 需要支持的文件格式（PDF、Word、网页等）

### 3. 核心功能需求
- 问答界面设计要点
- 引用来源展示方式
- 未覆盖问题的处理机制
- 知识库更新触发机制和频率

### 4. 评估方案
- 评估数据集：需要覆盖哪些场景（常见问题、边界问题、多轮对话）
- 评估指标：准确率、完整性、引用准确率
- 评估方式：人工评估标注规范、自动化评估方案

### 5. 风险和考量
- 幻觉风险及缓解措施（如：限制回答范围、添加免责声明）
- 知识库过期风险（更新机制）
- 数据安全（数据隔离、权限控制）
- 性能（大量并发时的响应时间保障）

### 6. 分阶段迭代计划
- Phase 1：最小可行性版本（覆盖 Top 50 常见问题）
- Phase 2：完整知识库覆盖 + 多轮对话
- Phase 3：主动推荐 + 多模态支持

请用清晰的产品经理语言，而非过于技术化的表达，确保非技术干系人也能理解。
```

## 模板 3：AI Agent / 智能助手 PRD

**使用场景**：设计一个能自主执行任务的 AI Agent（如：自动回复邮件、智能调度、自动化工作流）。

**提示词**：

```markdown
你是一位 AI Agent 产品的产品经理。请帮我设计一个 AI Agent 的 PRD。

## Agent 基本信息
- Agent 名称：[如：智能客服助手]
- 核心任务：[如：自动回复客户邮件咨询]
- 目标用户：[如：客服团队]
- 需要调用的工具/API：[如：查询订单 API、修改信息 API、发送邮件 API]

## 请包含以下内容：

### 1. Agent 能力定义
- 这个 Agent 能做什么（明确的能力边界）
- 这个 Agent **不能**做什么（明确限制）
- 决策逻辑：什么情况下 Agent 自主决策，什么情况下需要人类介入

### 2. 用户交互模式
- 触发方式（用户主动发起 / 系统自动触发 / 定时执行）
- 对话/交互流程（主流程 + 异常流程）
- 输出格式（文本回复 / 结构化数据 / 执行操作）

### 3. Agent 架构需求
- ReAct / Plan-and-Execute / 其他架构模式
- 工具/Function Calling 的定义规范
- 记忆机制（短期记忆 / 长期记忆 / 会话管理）
- 安全护栏（什么操作禁止执行、什么话题禁止讨论）

### 4. 评估方案
- 任务完成率：什么算一次成功的任务执行
- 安全性评估：不该做的事有没有做
- 效率评估：平均执行时间、工具调用次数
- 人工介入率：需要人工干预的比例

### 5. 风险和伦理考量
- 自主决策的风险边界（什么决策必须保留给人类）
- 错误操作的补救机制（撤销操作 / 错误回滚）
- 审计日志（记录每次决策和执行的完整链条）
- 用户透明性（用户是否知道他们在和 AI 交互）

### 6. 灰度到全量的上线计划
- 内部测试（仅团队可用）
- 有限用户测试（1% 流量）
- 逐步放量到全量
- 每个阶段的回滚条件和指标阈值
```

## 模板 4：内容生成类 AI PRD

**使用场景**：AI 写作助手、AI 设计生成、AI 视频/音频生成等内容创作类产品。

**提示词**：

```markdown
你是一位内容生成类 AI 产品的产品经理。请帮我写一份 PRD。

## 产品信息
- 产品类型：[如：AI 写作助手 / AI 图片生成 / AI 视频生成]
- 生成内容类型：[如：营销文案 / 产品图片 / 短视频]
- 目标用户和使用场景：[如：市场人员写营销文案、设计师做海报]

## 请输出 PRD，包含：

### 1. 产品定位
- 解决的问题和核心价值
- 与竞品的差异化定位
- 目标用户画像（至少 2 类典型用户）

### 2. 核心功能需求
- 输入方式（文本 Prompt / 模板选择 / 参数调节 / 参考图上传）
- 生成控制（风格、长度、温度等参数控制）
- 输出格式（文本 / 图片 / 音视频）
- 编辑能力（生成后的修改和调整）
- 批量生成 / 多版本生成

### 3. 质量要求
- 生成内容的质量标准（准确率 / 美观度 / 可用率）
- 不适合内容的过滤机制（安全审核、版权检查）
- 是否存在幻觉 / 不合理内容的处理

### 4. 模型选型考量
- 使用闭源 API 还是开源自部署
- 生成速度 vs 质量的权衡
- 成本模型：单次生成的预估成本

### 5. 评估指标
- 生成成功率
- 用户留存和使用频率
- 编辑修改率（用户需要修改多少）
- 内容通过安全审核的比例
- 用户满意度评分

### 6. 风险
- 版权问题（训练数据的版权、生成内容的版权归属）
- 安全合规（生成违法违规内容的过滤）
- 成本控制（防止 API 滥用和成本失控）

请在输出中特别注意 AI 生成内容的质量评估标准和风险控制方案。
```

## 模板 5：多模态 AI 产品 PRD

**使用场景**：结合视觉 + 文本 + 语音等多种模态的 AI 产品（如 AI 图片审核、AI 视频分析、视觉问答）。

**提示词**：

```markdown
你是一位多模态 AI 产品的产品经理。请帮我设计一份 PRD。

## 产品信息
- 产品名称：[填写]
- 核心能力：结合 [视觉 / 语音 / 文字] 的 [分类 / 生成 / 分析] 能力
- 使用场景：[如：用图片+文字描述进行商品搜索]

## 请输出 PRD，包含：

### 1. 多模态交互设计
- 用户输入方式（上传图片 + 文字提问 / 语音 + 视频 / 多选组合）
- 输出展示方式（图文结合 / 标注 + 分析结果 / 多模态结果）
- 交互流程：主路径和可能的用户体验问题点

### 2. 各模态的技术需求
- 视觉模型需求（分辨率、识别精度、支持的图片格式）
- 文本模型需求（语言支持、字数限制）
- 语音模型需求（如果有）（语种、口音、噪音环境）
- 多模态融合方式（早期融合 / 晚期融合 / 端到端多模态）

### 3. 产品功能清单
- 核心功能：必须有的能力
- 辅助功能：可选的增强能力
- 高级功能：未来迭代的愿景

### 4. 评估方案
- 每个模态的独立评估指标
- 多模态联合评估指标（如：图文匹配准确率）
- 端到端用户体验评估

### 5. 技术挑战和风险
- 多模态对齐问题
- 不同模态的延迟差异
- 训练数据的获取和标注成本
- 设备兼容性（移动端 vs 桌面端）

### 6. 竞品分析方向
- 列出主要竞品
- 从多模态能力、体验、成本三个维度做对比

请重点描述多模态融合场景下的用户体验设计考量。
```

## 模板 6：AI 功能优化 PRD（迭代型）

**使用场景**：对已有的 AI 功能做优化迭代（提升准确率、降低成本、改善体验），而非从 0 开始。

**提示词**：

```markdown
你是一位 AI PM，现在需要对一个已上线的 AI 功能做迭代优化。请帮我写一份迭代型 PRD。

## 现状描述
- 功能名称：[填写]
- 当前表现数据：[描述现有指标表现]
- 用户/业务反馈：[整理主要的反馈问题]
- 优化目标：[如：提升准确率 10%、降低延迟 50%]

## 请输出 PRD，包含：

### 1. 问题分析
- 当前功能的主要问题列表（附数据支持）
- 问题的根因分析（模型问题 / Prompt 问题 / 数据问题 / 评估问题）
- 用户受影响程度的量化描述

### 2. 优化方案
- 方案 A（推荐）：具体做什么改进
- 方案 B（备选）：替代方案
- 每个方案的预期效果和影响范围
- 不做的影响分析

### 3. 实验设计
- 评估方法（A/B 测试 / 离线评估 / 在线评估）
- 评估指标和验证方法
- 样本量预估和测试周期

### 4. 实施计划
- 所需的资源和团队
- 时间线（实验 → 灰度 → 全量）
- 回滚条件

### 5. 技术方案要点
- 需要修改的模块
- 数据回流需求
- 新增的监控指标

请以数据驱动的方式呈现，每个问题都有数据支撑，每个方案都有量化预期。
```

## 模板 7：AI 平台/中间件 PRD

**使用场景**：面向内部或外部开发者的 AI 平台、模型推理平台、Prompt 管理平台等基础设施类产品。

**提示词**：

```markdown
你是一位 AI 平台产品的产品经理，负责设计面向 [开发者 / 内部团队] 的 AI 平台。请帮我写一份 PRD。

## 平台定位
- 平台类型：[如：模型推理平台 / Prompt 管理平台 / AI 能力中台]
- 目标用户：[如：公司内部算法工程师 / 外部独立开发者 / 产品经理]
- 核心能力：[如：模型统一接入、Prompt 管理、效果监控]

## 请输出 PRD，包含：

### 1. 平台愿景和架构
- 平台解决的核⼼问题
- 在技术栈中的位置（上下游系统关系）
- 服务架构概览

### 2. 用户角色和权限
- 用户类型（管理员 / 普通用户 / 只读用户）
- 每种角色的权限范围
- 多租户需求（如果适用）

### 3. 核心模块功能
- 模块一：[如：模型管理 — 接入、部署、版本管理]
- 模块二：[如：Prompt 管理 — 编写、测试、版本、发布]
- 模块三：[如：评估中心 — 数据集管理、自动评估、人工评估]
- 模块四：[如：监控告警 — 调用量、成本、延迟、错误率]
- 每个模块的功能清单和使用场景

### 4. 体验需求
- 易用性：降低使用门槛的设计
- 文档和示例：API 文档、SDK、使用教程
- 开发者体验：调试工具、错误信息、日志

### 5. 平台指标
- 采用率（接入的产品数量 / API 调用量）
- 稳定性（SLA、可用性指标）
- 效率（模型部署时间、评估周期）
- 满意度（NPS 或用户调研分数）

### 6. 风险
- 单点故障风险
- 成本失控风险
- 平台依赖风险

请注重平台产品的"开发者体验"和"易用性"设计。
```

## PRD 写作技巧总结

| 技巧 | 说明 |
|------|------|
| **量化优先** | 每个需求都要有数据支撑，每个目标都要有指标 |
| **场景驱动** | 从用户场景出发，而非从技术能力出发 |
| **明确边界** | 清晰说明"做什么"和"不做什么" |
| **对齐语言** | 用团队能理解的语言 — 对算法说指标，对工程说架构，对业务说收益 |
| **评估前置** | 在 PRD 阶段就想清楚怎么评估，不要等到上线后再想 |
| **灰度思维** | 永远要有灰度计划和回滚方案 |
