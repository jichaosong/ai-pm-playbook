# 第 1 课：什么是 LLM？

> 本课目标：用最简单的语言理解大语言模型是什么、它怎么生成回答、它和数据库有什么不同，以及产品经理为什么需要理解它。

---

## 1. 简单定义

LLM 是 **Large Language Model**，中文叫**大语言模型**。

简单说：

> LLM 是一个读过大量文字的 AI，它学会了语言里的规律，然后根据你的问题生成回答。

它不是像人一样真正“理解世界”，更像一个非常强的自动补全系统。

例如你输入：

```text
中国的首都是
```

模型很可能继续生成：

```text
北京
```

你输入：

```text
帮我写一封请假邮件
```

模型会根据以前见过的大量邮件格式，生成一封看起来合理的邮件。

---

## 2. 生活类比

想象一个人读过很多书、网页、代码、邮件、客服对话和产品文档。

读得多了以后，他看到一句话的开头，就能猜后面应该怎么接。

LLM 做的事情，本质上也是：

> 根据前面的内容，预测后面最可能出现的文字。

---

## 3. 它不是数据库

很多初学者会以为：

> LLM 里面存着一张巨大的“问题-答案表”。

其实不是。

它更像是学会了很多语言和知识的规律。

比如它不一定存着某一句完整的退款政策，但它见过很多退款政策，所以知道退款政策一般会包含：

- 退款时间
- 退款条件
- 不可退款情况
- 联系方式
- 处理流程

所以你让它写退款政策，它能生成一份。

---

## 4. “大”是什么意思？

“大语言模型”里的“大”，通常是指模型有很多**参数**。

参数可以理解成：

> 模型内部很多很多可以调整的小旋钮。

训练模型时，这些小旋钮会不断被调整，让模型越来越会预测文字。

一个模型可能有：

```text
7B 参数 = 70 亿个参数
```

也可能有：

```text
70B 参数 = 700 亿个参数
```

但注意：

> 模型越大，不一定越适合你。

大模型通常更强，但也更贵、更慢、更难部署。

小模型如果数据好、任务明确，也可以非常实用。

---

## 5. LLM 怎么回答问题？

非常简化的流程是：

```text
你的文字
→ 切成 token
→ token 变成数字
→ 模型处理这些数字
→ 预测下一个 token
→ 不断重复
→ 生成完整回答
```

也就是说：

> 模型不是直接“看懂中文”，而是把文字变成数字，再做数学计算。

后面我们会详细讲：

- token 是什么
- tokenization 是什么
- embedding 是什么
- transformer 是什么
- attention 是什么

---

## 6. 它能做什么？

LLM 可以用来：

- 回答问题
- 写文章
- 写代码
- 总结文档
- 翻译
- 做客服
- 做 AI Copilot
- 做自动化工作流

但它不是万能的。

它容易在这些场景出问题：

- 事实要求很高的问题
- 需要最新信息的问题
- 企业内部私有知识
- 多步骤严谨推理
- 需要调用真实系统的任务

所以 AI 产品设计不是简单地“接一个模型 API”。

真正的 AI 产品通常还需要：

- 数据
- 检索
- 工具
- 权限
- 评估
- 监控
- 成本控制
- 人工兜底

---

## 7. 对产品经理有什么用？

产品经理不需要一开始就会训练模型，但需要理解模型能力边界。

你需要知道：

| 问题 | 为什么重要 |
|------|-----------|
| 模型适合做这个任务吗？ | 避免为了 AI 而 AI |
| 需要 RAG 还是微调？ | 影响方案复杂度和成本 |
| 输出是否可以被验证？ | 影响上线风险 |
| 用户能不能接受错误？ | 影响交互和兜底设计 |
| 成本是否可控？ | 影响商业模式 |

如果你不知道 LLM 的基本工作方式，就很容易把 AI 产品设计成“看起来很酷，但上线后不稳定”的功能。

---

## 8. 本课总结

LLM 是一个通过大量文字训练出来的 AI 模型。

它的核心能力是：

> 根据上下文，预测接下来最可能出现的文字。

它不是普通数据库，也不是人类大脑。

它是一个强大的语言模式生成器。

---

## 9. 心智模型

把 LLM 想成：

> 一个非常博学、非常会接话的助手。

你给它：

```text
任务 + 背景 + 要求
```

它就根据学过的模式生成回答。

输入越清楚，输出通常越好。

---

## 10. 初学者常见错误

### 错误 1：以为 AI 永远是对的

LLM 会犯错，而且有时会很自信地犯错。

这叫**幻觉**，也就是 AI 编了一个看起来很真的答案。

### 错误 2：以为模型越大越好

不一定。

实际项目里要考虑：

- 成本
- 速度
- 准确率
- 部署难度
- 是否能本地运行
- 是否适合你的业务场景

### 错误 3：把提示词和微调混为一谈

**提示词 Prompting**：通过更好的指令，让模型回答得更好。

**微调 Fine-tuning**：用新的训练数据，改变模型的行为习惯。

### 错误 4：以为模型天然懂你的业务

模型不一定知道你的公司、产品、客户和内部文档。

要让它懂你的业务，通常需要：

- 把资料放进 prompt
- 做 RAG
- 做微调
- 接工具
- 建记忆系统

---

## 11. 小练习

问任意一个 AI：

```text
用“餐厅”的比喻解释什么是大语言模型。
```

然后再问：

```text
用“老师”的比喻解释什么是大语言模型。
```

比较两个回答的不同。

你会发现：

> 同一个概念，可以通过不同 prompt 得到不同风格的解释。

---

## 12. 小项目

创建一个“LLM 概念解释器 prompt”：

```text
你是一个耐心的老师。
解释 [主题]，要求：
1. 一个简单定义
2. 一个生活类比
3. 一个例子
4. 一个初学者误区
5. 一个小练习

主题：大语言模型
```

然后把主题换成：

```text
token
embedding
fine-tuning
RAG
agent
```

练习目标：

> 学会用稳定结构，让模型稳定输出你想要的解释。
