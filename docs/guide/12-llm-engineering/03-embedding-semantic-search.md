# 第 3 课：Embedding 和语义搜索

> 本课目标：理解 embedding 为什么能把文字变成“意思坐标”，以及它如何支撑语义搜索、RAG、推荐、相似问题匹配和 AI 记忆。

---

## 1. 什么是 embedding？

上一课我们讲了：

```text
文字 → token → 数字 ID
```

但这里有一个问题。

假设：

```text
猫 → 1024
狗 → 9876
苹果 → 456
```

这些数字 ID 本身没有“意思”。

模型不能只靠：

```text
猫 = 1024
狗 = 9876
```

就知道猫和狗都属于动物。

所以模型还需要一种方式，把文字变成带有“意思”的数字。

这就是：

> Embedding，向量嵌入。

简单说：

> Embedding 是把文字变成一串数字，用这些数字表示文字的意思。

比如：

```text
猫
```

可能被变成：

```text
[0.21, -0.44, 0.78, 0.09, ...]
```

这串数字叫：

> 向量。

你可以先把向量理解成：

> 一组数字坐标。

---

## 2. 生活类比：意思地图

假设我们有一张“意思地图”。

在这张地图上：

- 猫和狗离得近
- 猫和汽车离得远
- 北京和上海离得近
- 北京和香蕉离得远
- 退款和售后离得近
- 退款和火锅离得远

Embedding 的作用就是：

> 把每段文字放到“意思地图”上的某个位置。

如果两个词意思接近，它们在地图上距离就近。

如果意思差很多，距离就远。

---

## 3. 例子：语义接近

比如这些词：

```text
猫
狗
兔子
老虎
```

它们都和动物有关，所以在 embedding 空间里通常比较接近。

这些词：

```text
汽车
地铁
飞机
公交车
```

它们都和交通有关，也会比较接近。

但：

```text
猫
```

和：

```text
发票
```

通常距离很远。

因为它们语义关系弱。

---

## 4. 为什么 embedding 很重要？

Embedding 是很多 AI 系统的基础。

它用于：

- 模型理解输入
- 语义搜索
- RAG
- 推荐系统
- 相似问题匹配
- 文档检索
- 用户画像
- AI 记忆系统

尤其是 RAG 里，embedding 非常关键。

---

## 5. 普通搜索 vs 语义搜索

我们先看普通搜索。

你搜索：

```text
如何退款？
```

普通关键词搜索可能只找包含“退款”两个字的内容。

但如果文档写的是：

```text
如何申请退货并拿回款项？
```

它可能找不到，因为没有直接出现“退款”。

语义搜索不一样。

它会知道：

```text
退款
```

和：

```text
拿回款项
```

意思很接近。

所以它能找到相关内容。

这就是 embedding 的价值。

---

## 6. 生活类比：找同义表达

假设你开了一家店。

客户问：

```text
我怎么退款？
```

但你的客服文档写的是：

```text
用户可以在订单详情页申请售后，审核通过后款项会原路返回。
```

关键词“退款”不一定完全匹配。

但意思是相关的。

Embedding 可以帮助系统判断：

> 用户问的“退款”和文档里的“款项原路返回”很接近。

所以它能把这段文档找出来。

---

## 7. 什么是向量数据库？

Embedding 生成后，通常会存进一个地方。

这个地方叫：

> 向量数据库，Vector Database。

它的作用是：

> 存放很多 embedding，并快速找出意思最接近的内容。

你可以把它想成：

> 一个按“意思距离”查找的仓库。

普通数据库擅长查：

```text
用户 ID = 123
订单状态 = paid
创建时间 > 2026-01-01
```

向量数据库擅长查：

```text
哪几段文字和这个问题意思最接近？
```

---

## 8. Embedding 在 RAG 里的作用

RAG 的大概流程是：

```text
用户问题
→ 变成 embedding
→ 去向量数据库找相似文档
→ 把找到的文档给 LLM
→ LLM 根据文档回答
```

举个例子。

用户问：

```text
超过 7 天还能退款吗？
```

系统会：

1. 把这个问题变成 embedding
2. 去知识库里找语义最接近的文档
3. 找到退款政策相关内容
4. 把内容放进 prompt
5. 让模型根据内容回答

这样模型不是凭空回答，而是基于你的资料回答。

这能减少幻觉。

---

## 9. Embedding 和 LLM 是一回事吗？

不是。

它们相关，但不一样。

| 概念 | 作用 | 类比 |
|---|---|---|
| LLM | 生成回答 | 会说话的老师 |
| Embedding 模型 | 把文字变成意思坐标 | 图书馆分类员 |
| 向量数据库 | 存储并查找相似内容 | 按主题找书的图书馆 |

很多 AI 产品会同时用它们。

比如 AI 客服：

```text
用户问题
→ embedding 搜索知识库
→ 找到相关文档
→ LLM 生成自然语言回答
```

## 10. Embedding 是怎么表示“意思”的？

你不需要一开始就理解所有数学。

先用一个简单例子。

假设我们只用 2 个维度：

```text
[动物程度, 交通程度]
```

那么：

```text
猫     → [0.95, 0.01]
狗     → [0.93, 0.02]
汽车   → [0.02, 0.96]
飞机   → [0.01, 0.92]
```

猫和狗接近，因为它们都是动物。

汽车和飞机接近，因为它们都和交通有关。

真实 embedding 不是 2 维，可能是几百维、上千维。

但心智模型一样：

> 每个维度都在帮助表达某种隐藏的语义特征。

---

## 11. 相似度是什么？

当我们有两个 embedding，就可以计算它们有多像。

常见方法叫：

> cosine similarity，余弦相似度。

你不需要先记数学。

简单理解：

> 它用来判断两个向量方向是否接近。

方向越接近，意思通常越接近。

比如：

```text
“如何退款？”
```

和：

```text
“怎么申请退钱？”
```

相似度可能很高。

而：

```text
“如何退款？”
```

和：

```text
“今天晚饭吃什么？”
```

相似度会很低。

---

## 12. 小代码示例：相似度思想

下面不是完整 embedding 模型，只是帮你理解“距离”的概念。

```python
import math

def cosine_similarity(a, b):
    dot = sum(x * y for x, y in zip(a, b))
    norm_a = math.sqrt(sum(x * x for x in a))
    norm_b = math.sqrt(sum(y * y for y in b))
    return dot / (norm_a * norm_b)

cat = [0.95, 0.01]
dog = [0.93, 0.02]
car = [0.02, 0.96]

print(cosine_similarity(cat, dog))  # 高，相似
print(cosine_similarity(cat, car))  # 低，不相似
```

这段代码表达的核心思想是：

> 把意思变成数字后，就可以用数学找相似内容。

## 13. Embedding 能解决什么实际问题？

### 场景 1：智能客服

用户问：

```text
为什么我的钱还没退回来？
```

系统能找到：

```text
退款到账时间通常为 3-7 个工作日。
```

即使用户没说“到账时间”，embedding 也可能找到相关内容。

### 场景 2：企业知识库

员工问：

```text
请假要走什么流程？
```

系统找到 HR 文档里的：

```text
员工需在系统内提交休假申请，经直属主管审批后生效。
```

### 场景 3：相似问题推荐

用户问：

```text
账号无法登录怎么办？
```

系统推荐相似问题：

```text
忘记密码怎么办？
验证码收不到怎么办？
账号被冻结怎么办？
```

### 场景 4：AI 记忆

你告诉 AI：

```text
我喜欢简洁的产品方案。
```

系统可以把这句话存成 embedding。

以后你问：

```text
帮我写一个需求方案
```

AI 可以找回这条记忆，然后用更简洁的风格回答。

## 14. Embedding 的限制

Embedding 很有用，但不是魔法。

它可能出错。

### 问题 1：相似不等于正确

两个句子意思接近，不代表答案一定正确。

```text
退款规则
```

和：

```text
退货规则
```

可能接近，但业务上可能不是一回事。

### 问题 2：细节可能丢失

Embedding 很擅长抓大意，但对精确细节不一定完美。

比如：

```text
7 天内可退款
```

和：

```text
7 天后不可退款
```

有时语义很接近，但结论相反。

所以 RAG 不能只靠 embedding，还要做：

- reranking
- 精准关键词过滤
- 权限控制
- 文档版本管理
- 答案引用

### 问题 3：垃圾数据会影响效果

如果你的知识库里有很多：

- 过期文档
- 重复文档
- 错误文档
- 格式混乱
- 标题不清楚

embedding 搜索也会变差。

AI 系统质量经常不是模型问题，而是数据问题。

## 15. Embedding 和产品经理的关系

如果你做 AI 产品，embedding 会影响很多设计问题。

你需要思考：

### 搜索体验

用户问法很多，系统能不能找对资料？

### 知识库建设

文档是否结构清楚？
标题是否准确？
有没有过期内容？

### 权限

不同员工能不能搜到不同文档？

### 解释性

AI 回答时能不能告诉用户：

> 我参考了哪几篇文档？

### 质量评估

用户问 100 个问题，系统能找对多少次？

这叫检索准确率，后面讲 RAG 评估时会深入讲。

## 16. Embedding、Token、Context 的关系

把前三个概念串起来：

| 概念 | 简单解释 | 类比 |
|---|---|---|
| Token | 模型读文字的小块 | 一口食物 |
| Context window | 模型一次能看的 token 总量 | 桌面大小 |
| Embedding | 文字的意思坐标 | 地图位置 |

完整流程可以这样想：

```text
文字先被切成 token
token 变成数字
模型用这些数字理解上下文
embedding 让系统能按意思找相似内容
```

## 17. 本课总结

Embedding 是把文字变成一串数字，用数字表示文字的意思。

它让 AI 可以做：

- 语义搜索
- RAG 检索
- 文档匹配
- 推荐
- 相似问题查找
- AI 记忆

它的核心价值是：

> 不只看字面一样，而是看意思接近。

## 18. 心智模型

把 embedding 想成：

> 把每句话放到一张“意思地图”上。

意思相近的句子离得近。

意思不同的句子离得远。

向量数据库就是这张地图的搜索系统。

## 19. 初学者常见错误

### 错误 1：以为 embedding 会真正理解一切

Embedding 是相似度工具，不是真正的人类理解。

它能帮你找相关内容，但不能保证一定正确。

### 错误 2：只用 embedding，不看业务规则

比如退款政策里：

```text
7 天内可退
```

和：

```text
超过 7 天不可退
```

业务结果完全不同。

系统需要结合精确规则判断。

### 错误 3：知识库不清洗，直接做 RAG

如果文档本身很乱，embedding 再好也会找错。

垃圾进，垃圾出。

### 错误 4：忽略权限

企业知识库里，不是所有人都能看所有文档。

RAG 系统必须做权限控制。

## 20. 小练习

看下面几句话，判断哪些意思更接近：

```text
A: 我想申请退款
B: 怎么把钱退回来？
C: 我想修改收货地址
D: 售后款项什么时候到账？
```

你可以这样判断：

- A 和 B 很接近
- A 和 D 有关系，但 D 更关注到账时间
- C 和它们关系较弱

这就是 embedding 搜索想做的事情。

## 21. 小项目

设计一个“客服知识库语义搜索”的小方案。

主题：电商退款。

准备 5 条知识：

```text
1. 用户可在订单完成后 7 天内申请退款。
2. 退款审核通过后，款项将在 3-7 个工作日内原路返回。
3. 生鲜类商品签收后不支持无理由退款。
4. 若商品破损，请上传照片并联系客服处理。
5. 优惠券支付部分不支持现金退回。
```

然后写 3 个用户问题：

```text
1. 我的钱什么时候到账？
2. 收到的水果坏了怎么办？
3. 优惠券能退成现金吗？
```

练习目标：

> 判断每个问题应该匹配哪条知识。

答案大致是：

```text
1 → 第 2 条
2 → 第 4 条
3 → 第 5 条
```
